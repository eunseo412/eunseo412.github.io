# c++ 문법
## else if
- else if문을 사용하여 한가지 조건만 실행되도록 만들 수 있다.
- else if문을 사용하면, 어떤 else if문에서 참이 나온 경우, 다른 else if 문은 생략된다.
- if문 여러개를 사용하면, 참이든 아니든 모든 조건을 체크한다.
  
```cpp
int main()
{
    int input = 4;
    if (input == 3)
    {
        int a = 1;
    }
    else if (input == 4)
    {
        int a = 0;
    }
    else if (input == 5)
    {
        //
    }
    else
    {
        //
    }
    return 0;
}
```
- 디버깅 해보면, if문과 첫번째 else if문만 실행됨을 확인할 수 있다.

## %연산자 활용 = 짝수 세기
- % 연산자를 활용하면 나머지를 구할 수 있다.
```cpp
int main()
{
    int count = 0;
    int arr[5] = {1, 2, 3, 4, 5};

    for (int i = 0; i < 5; i++)
    {
        if (arr[i] % 2 == 0)
        {
            count++;
        }
    }
}
```


# Jekyll GitHub 블로그에 Tag 기능 추가하기 (Minima Theme)
### Jekyll 글에 태그 추가
포스트의 Front Matter에 `tags`를 추가하면 된다.
```html
---
layout: post
title: "TIL"
tags: [C++, TIL]
---
```

### .../_layout/post.html 생성
- `theme: minima`를 사용하는 경우 `_layouts` 폴더가 기본적으로 보이지 않는다.
- 직접 `_layouts` 폴더를 생성한다.
- 해당 폴더에 post.html 를 생성한다.
- post.html 파일에 태그표시를 만들어준다.

### 태그 페이지 만들기
- 루트 디렉토리에 `tags.md` 생성한다.
- 태그의 목록, 해당 태그의 글목록을 볼 수 있는 페이지를 생성한다.

# 오늘의 삽질
## Git 충돌

GitHub에 push할 때 다음 오류가 발생했다.

```
! [rejected] main -> main (non-fast-forward)
```

원인: remote repository가 로컬보다 앞서 있는 상태

#### 해결 방법
`git pull origin main --rebase`
- 내 작업을 최신 main 위에서 한 것처럼 히스토리를 재정렬할 수 있다.

#### --rebase의 작동 방식
- origin/main 최신 커밋을 가져온다.
- 내 커밋을 잠깐 분리한다.
- 최신 커밋 뒤에 내 커밋을 붙인다.
  
그 이후에 push 하면 된다.
`git push origin main`

## jekyll로 구현한 페이지에 liquid 코드 삽입 시 오류
- 코드 전체를 백틱3개(```)으로 감싸도, 코드에 문법적인 오류가 있으면 에러날 수 있다.
- 코드를 {% raw $}, {% endraw %}로 감싸는게 가장 안전하다.
- 특히 백틱1개(`)로 감싸는 경우 쉽게 에러난다.
