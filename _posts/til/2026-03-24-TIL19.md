- weight의 사이즈: output 종류의 개수 * input 데이터의 크기
- input data를 쭉 펼쳐서 수의 나열로 생각

2. Visual(?) viewpoint
- 커널을 template으로 생각
- weight와 input을 곱한다.
- 학습 distribution을 최대한 반영한 종류별 하나의 template 존재.
- 곱했을 때 값이 커지면 커질수록 cosine similarity 가 크다.
- 큰 값을 가지는 class가 출력 도메인에서 선택된다.

# loss function
- 현재 모델이 얼마나 좋은지 알려준다.
- single example의 loss는 L(f(xi, W), y)
- 모델의 loss는 single example의 loss값을 다 더해서 평균낸 값

### Hinge Loss
- 정답이 아닌 다른 클래스에 대해 각각 계산 수행하고 더한다.
- 다른 클래스 점수 - 추측한 클래스 점수 + 1 의 값 vs 0 을 비교한다.
