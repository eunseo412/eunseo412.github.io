# LONGAGENT 논문 핵심

## 목적: LLM with long context 문제 해결
- expensive training 비용
- high inference latency
- lost in the middle

## 해결: LONG AGENT(LLaMA-7B(4K) 기반)
- multi-agent collaboration
- 4K context-window를 가진 base-model에서 출발
  
- leader : user intent 이해, hallucination 피해서 정확한 information 모으기
- team-member : document에서 information 수집
- inter-member communication : 정보 공유를 통해 응답 충돌(hallucination이 원인)을 해결

## 추론 과정
- expert models 생성, Member selection
  - QA experts, retrieval experts, and mathematical experts
  - 리더가 task description(d)에 적합한 expert 모델을 선택
  - long text (x) 를 n개의 chunks {c1, c2, ..., cn}로 쪼갬
  - n members {m1, m2, ..., mn}에게 각 chunk 부여
  
- Collaborative Reasoning
  - query(q)를 multiple sub-questions로 쪼개기
  - a ∼ Leader(a|S, q)
  - S= {s1, s2, ..., sm} : historical dialogue states
  - si : 한 라운드에서의 리더의 instruction, 모든 멤버의 responses
  - a ∈{NEW_STATE, CONFLICT, ANSWER}
   
- Resolving Conflicts: 오류를 내던 agent도 정답이 있는 chunk가 입력되면 틀렸던 답을 고친다는 아이디어
  - hallucination= mi(ci) 
  - Truth= mj (cj) 
  - Truth= mj (cj ⊕ci)
  

## 평가 task:
Needle-in-a-Haystack PLUS
long-text retrieval
numerical comparison

- 리더의 instruction에 관한 정보가 없는 chunk가 부여된 member의 답변 최적화
문제: chunk에 답이 없으면 internal data 기반으로 틀린 답
논문에서 해결책: reject 비율 조정 → reject 비율 올라갈수록 correctly refusing 비율 상승
BUT 동시에 accuracy 감소. document에 정답이 있는데도 답을 안하는 case가 발생하기 때문이다.
해결: 적당한 reject비율 찾기, 필요없는 chunk는 member에게 배분하지 않기
