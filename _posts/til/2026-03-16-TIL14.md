## construction and destruction
- class instance를 만들 때 constructor가 실행되기 전에, 
- member변수(private에서 요구되는 또다른 instance or 정수 or 뭐든..)생성부터 진행된다.
- member변수가 또다른 clas의 instance라면 그 class의 instance 생성부터 진행된다.

- member변수 만들때 필요한 class에 attribute가 요구 될때 -> initializer list(constructor 부분)에 값을 줘야한다.
- private 구간은 이미 만들어진 틀이기 때문에 따로 데이터를 넣는건 이상하다.
- 여러개 요구될 때 -> constructor 부분에 instance명과 함께 attribute를 넣어주는데, 그걸 여러개 써야할 때 순서 바꾸면 에러난다.


```cpp
class Test
{
public:
    Test() : a(10), b(a)
    // Test() : b(a), a(10) 해도 error 안 남
    {
    }

private:
    int a;
    int b;
};

int main()
{
    Test inst_a;
}
```

## 여러개의 constructor
- 변수 개수에 맞는 constructor 선택해서 실행된다.
- function overloading이라고 한다.
- parameter 안 받을 때 -> default constructor

```cpp

class Constipated
{
public:
    Constipated()
    {
        cout << "default constructor" << endl;
        name_ = "000";
        days_ = 0;
    }
    Constipated(string name)
    {
        cout << " constructor 1" << endl;
        name_ = name;
        days_ = 0;
    }
    Constipated(string name, int days)
    {
        cout << "constructor 2" << endl;
        name_ = name;
        days_ = days;
    }

private:
    string name_;
    int days_;
};

int main()
{
    Constipated t;
    Constipated a("Kim");
    Constipated b("Choi", 6);
}
```

## pointer
- 포인터는 단순히 주소를 저장하는 역할.
- 특별한 역할 하나가 있다면, *을 찍어줬을 때 그 주소값에 해당하는 데이터에 접근할 수 있다는 점이다.

```cpp
int main()
{
    int a = 5;
    int *p = &a;
    cout << a << *p << endl;
    }
```
- 포인터의 타입을 저장할 변수의 타입과 일치해야함을 잊지말자

# 오늘의 Q&A
## 1. void의 사용
### void 함수
- 함수형은 return문에서 반환해 주는 값의 데이터형과 일치해야 한다. 
- 만약 반환값이 없으면 함수형을 void로 지정한다.
- void형 함수 내에서 식이 없는 return문을 작성할 수 있다.
- return을 만나면 현재의 실행 위치와 관계없이 함수가 마무리된다.

### 인수 없음의 의미
- void는 함수를 정의할 때 인수가 없다는 의미로도 사용할 수 있다. 
- 인수가 없는 함수를 정의하는 경우, 괄호 안을 비워도 되지만 void를 작성하는게 바람직하다.

### void 변수
- void 변수는 생성 불가능하다.
- 변수의 타입을 모르기때문에 메모리를 얼마나 할당할지 결정할 수 없다.

### void 포인터
- void 포인터는 가능하다.
- 포인터는 타입에 관계없이 고정된 크기만큼 할당되기 때문이다.
- 아래 코드에서 void로 만들어진 포인터 a가 num의 주소를 잘 가지고 있음을 확인할 수 있다.


```cpp
#include <iostream>
using namespace std;

int main() {
	int num = 3;
	void *a;
	a = &num;

	cout << &num << endl;
	cout << a << endl;

	return 0;
}
```
- 그러나 주소로부터 값을 읽지는 못한다.
- a에 저장된 시작주소에서부터 얼만큼 읽어야할지 모르기 때문이다.
```cpp
#include <iostream>
using namespace std;

int main() {
	int num = 3;
	void *a;
	a = &num;

	cout << *a << endl;

	return 0;
}
```

- 아래와 같이 void형 포인터를 정수형 포인터로 변환해주면 주소로 값을 확인할 수 있다.
- a가 int를 가리키는 포인터임을 알려주는 것이다. -> 시작주소부터 4byte를 읽는다.
  
```cpp
#include <iostream>
using namespace std;

int main() {
	int num = 3;
	void *a;
	a = &num;

	cout << *(int *)a << endl;

	return 0;
}
```


## 2. main 함수의 인자
- main 함수는 사실 인자를 가지고 있다.
- argc는 인자의 개수, agrv는 각각의 인자들이다.
- 기본적으로 파일의 실행경로를 인자로 받는다.
```cpp
int main(int argc, char **argv)
```
```cpp
#include <iostream>
using namespace std;

int main(int argc, char **argv) {
	cout << "받은 인자의 개수: " << argc << endl;
	for (int i = 0; i < argc; i++) {
		cout << "프로그램이 받은 인자: " << argv[i] << endl;
	}

	return 0;
}
```

## 3. 함수의 작성

- main 함수 아래에 함수를 작성하고 main함수 위에 선언부만 넣어도 된다.

## 4. 정수가 4바이트인 이유
- 대부분의 CPU가 32비트 단위로 데이터를 처리하도록 설계되어 있어 4바이트 정수가 계산에 효율적이다.
- 32비트 정수는 약 ±21억 범위를 표현할 수 있어 일반적인 프로그램에서 충분한 범위를 제공한다.
